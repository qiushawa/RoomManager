Admin Guide

管理員後台操作指南

本文件說明後台主要功能與操作限制，供日常維運使用。

返回文件首頁 使用者導覽

適用對象

  • 具備管理員帳號的系統維運人員
  • 負責借用審核、教室資料管理、黑名單管理與長期借用設定的人員

後台入口與登入

  1. 開啟 /admin/login
  2. 輸入管理員帳號與密碼
  3. 登入後系統會導向「總覽」頁

開發測試環境預設帳號：帳號 test、密碼 test。

重要前置條件

系統要求目前或未來至少要有一筆學期資料，否則後台多數功能會被限制。
  • 可進入：登入、系統設定
  • 會被導回系統設定：總覽、審核、教室管理、黑名單、長期借用等功能

側欄功能總覽

  • 總覽
  • 審核列表
  • 短期借用紀錄
  • 教室管理
  • 黑名單管理
  • 長期借用管理
  • 系統設定

1. 系統設定（首次必做）

  • 新增學期（學年、學期、開始日、結束日）
  • 檢視既有學期清單

驗證規則：學年 + 學期不可重複、日期區間不可重疊、結束日需大於或等於開始日。

2. 總覽

顯示關鍵統計與圖表，如啟用教室數、借用總筆數、待審核筆數、熱門時段等。

3. 審核列表

  • 關鍵字搜尋（教室代碼、借用人、借用事由）
  • 更新狀態為已核准、已駁回、已取消
  • 通知鈴鐺可快速查看待審核申請數

4. 短期借用紀錄

顯示所有非待審核案件，支援狀態篩選與關鍵字搜尋。

5. 教室管理

  • 新增教室、啟用/停用教室、刪除教室
  • 教室代碼不可重複，長度上限 7 字元
  • 教室名稱長度上限 25 字元
  • 已有借用紀錄的教室不可刪除

6. 黑名單管理

  • 可新增或更新停權資料
  • 需填學號、黑名單原因（可複選）、停權結束日
  • 同一借用人再次送出會更新既有資料

7. 長期借用管理

7.1 手動新增

建立固定週期借用記錄，提交前會檢查與課表/既有資料衝突。

7.2 教室課表匯入

  • 先預覽再確認匯入
  • 一次僅能匯入同一大樓（CB、GC、RA）
  • 同學期同教室匯入資料會先清除再重建
  • 可對單一教室撤回匯入

7.3 已儲存記錄

可檢視本學期手動新增記錄，並執行撤回。

常見問題排除

  1. 一直被導回系統設定：目前與未來都沒有學期資料，請先新增學期。
  2. 匯入課表顯示找不到學期：請先建立對應學年/學期。
  3. 教室不能刪除：已有借用紀錄，建議改為停用。
  4. 黑名單新增失敗：學號不存在借用人資料，請先確認來源。
